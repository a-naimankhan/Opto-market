Back-End Requirements — Django + DRF

All Backend Requirements Covered

🗂
Models (≥4 required — 6 implemented)
  • UserProfile — extends User with role (buyer/seller)
  • Product — name, price, discount_price, description,
    image, stock_quantity, category (FK), owner (FK)
  • Category — name (unique)
  • Order — customer (FK), product (FK), status, total_price,
    delivery & payment method, recipient info
  • ProductReview — product (FK), user (FK), rating, comment
  • OrderStatusHistory — audit trail of status changes
  • ForeignKey ①: Product → Category  |  ForeignKey ②: Product → User
  • ForeignKey ③: Order → User  |  ForeignKey ④: Order → Product
🔑
Custom Model Manager
  • AvailableProductManager — filters queryset to only products with stock_quantity > 0
  • Used on the public catalogue endpoint (buyers see only in-stock items)
  • Seller's own "my products" view uses the default manager (sees all)
📦
Serializers
  • ModelSerializer ① ProductSerializer (all fields, read-only owner)
  • ModelSerializer ② OrderSerializer (nested ProductSerializer)
  • ModelSerializer ③ CategorySerializer
  • ModelSerializer ④ ProductReviewSerializer (username via source)
  • Serializer ① LoginSerializer (plain, write-only password)
  • Serializer ② OrderSummarySerializer (stats response)
🌐
Views & Authentication
  • CBV (APIView) ① ProductList — GET/POST (full CRUD entry)
  • CBV (APIView) ② ProductDetail — GET/PATCH/DELETE
  • CBV (APIView) ③ CategoryList / CategoryDetail
  • CBV (APIView) ④ OrderList — GET seller orders / POST new
  • CBV (APIView) ⑤ OrderStatusUpdate, ProductReviewList
  • FBV ① register_user — @api_view POST, AllowAny
  • FBV ② logout_user — @api_view POST, token blacklist
  • FBV ③ user_profile — @api_view GET
  • FBV ④ price_stats — @api_view GET (Avg/Min/Max)
  • JWT: TokenObtainPairView + TokenRefreshView + blacklist on logout
  • CORS: django-cors-headers, CORS_ALLOW_ALL_ORIGINS = True
  • owner=request.user on Product.save(); customer=request.user on Order

Front-End Requirements — Angular 17

All Angular Requirements Covered

🔀
Routing — 13 Named Routes (≥3 required)
  • / & /buyer — public product catalogue (Landing)
  • /login — JWT login page
  • /buyer/cart — cart (guarded: buyerAuthGuard)
  • /buyer/orders — order history
  • /buyer/checkout — checkout form
  • /seller/products — CRUD product management
  • /seller/categories — category management
  • /seller/order — incoming orders
  • /seller/customer — customer list
  • /seller/reviews — seller reads product reviews
  • Route guards: sellerAuthGuard + buyerAuthGuard
🔒
JWT Authentication
  • HTTP Interceptor (auth-interceptor.ts) — clones every outgoing request and sets Authorization: Bearer <token> header from localStorage
  • Login page collects username + password, calls /api/token/, stores access & refresh tokens
  • Logout calls /api/logout/ (blacklists refresh token) then clears localStorage
  • Role-based routing: seller → /seller panel, buyer → / catalogue
⚠️
Error Handling
  • All HTTP calls use .subscribe({ error: … }) to catch failures
  • Error messages shown in template via *ngIf / @if on error string
  • Discount validation: warns if discount ≥ regular price
  • Form validation prevents empty required fields before API call
🖱
Click Events triggering API (≥4 required)
  • (click) ① Save product → POST/PATCH /api/products/
  • (click) ② Delete product → DELETE /api/products/:id/
  • (click) ③ Filter by category → GET /api/products/?category=n
  • (click) ④ Place order (checkout) → POST /api/orders/
  • (click) ⑤ Update order status → PATCH /api/orders/:id/status/
  • (click) ⑥ Clear discount → PATCH /api/products/:id/ (discount_price: null)
  • (click) ⑦ Submit review → POST /api/products/:id/reviews/
📝
Form Controls ([(ngModel)]) — ≥4 required
  • ngModel ① Login: username field
  • ngModel ② Login: password field
  • ngModel ③ Product form: name, price, discount_price, description, category, unit, min_quantity, stock_quantity, seller_name, seller_phone
  • ngModel ④ Checkout form: firstName, lastName, phone, email, delivery method, payment method, recipient fields
  • ngModel ⑤ Review form: rating, comment
🔁
@for / @if — Angular 17+ Control Flow
  • @for — product card list, order list, review list, category buttons, cart items, status history logs
  • @if — discount price block, empty-state messages, loading spinners, seller/buyer conditional UI, form error messages, recipient section toggle
  • Services: ProductService, AuthService, OrderService, CartService — all inject HttpClient
  • Interfaces: Product, Category, Order, ProductReview, CartItem in api.models.ts
